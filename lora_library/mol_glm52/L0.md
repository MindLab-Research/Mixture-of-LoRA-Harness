# L0 chat

## Summary
General-purpose chat and direct tool-assistant work. This includes explanation and editing as well as straightforward web/current-information lookup, generic product or place search, browser and calendar actions, reminders, routes, shareable lists/pages, and small direct file or integration actions. Choose L0 only when the request clearly matches this family; ambiguous or underspecified requests default to L2.

## Router Line
clearly specified general-purpose chat and direct tool-assistant requests: explain, edit, summarize, recommend, or perform a straightforward web/current-info lookup, generic product/place search, browser action, reminder, calendar lookup, route lookup, shareable list/page, or one-step app operation; a small direct file/browser/integration action also stays L0 unless it is a software benchmark, repository, or engineering workflow; choose L0 only for clearly general-purpose requests, not as the fallback
Boundary: personalized meal or diet planning, food logging, calendar or reminder actions, and a one-step temporary sandbox, notebook, command, or small file integration stay L0 even with group, location, health, or budget details; only a repository, software benchmark, or engineering workflow is L2.
Boundary: asking the assistant/model to identify itself, or correcting a claim about its official name, parameter count, base model, LoRA/expert composition, architecture, or post-training foundation, is clearly L0 even with technical model terminology; it becomes L2 only when the requested deliverable is code, a repository change, terminal execution, or code-level engineering analysis.
Boundary: a vague or underspecified request without a concrete general-purpose goal is not L0 and defaults to L2.
Boundary: a request that only asks to handle, continue, fix, or proceed without identifying an object or concrete goal is underspecified and defaults to L2.

## Description
Entry/router LoRA for open-domain conversation and the broad general-tool task family, but not the target fallback. Personal context such as food, health, travel, family, budget, or location does not by itself make a direct search, calendar, logging, browser, or simple app action an L1 request. Technical discussion stays on L0 when it asks for concepts, comparisons, reports, or guidance rather than a repository or software-benchmark workflow; ambiguous requests target L2.

## Routing Rules
- Start every request on L0.
- Keep decoding on L0 for ordinary chat, language polishing, translation, conceptual explanation, simple advice, or ambiguity.
- Keep decoding on L0 for general technical concepts and ML/system discussions such as KV cache, attention, quantization, LoRA fine-tuning, prompt engineering, routing benchmarks, or model-evaluation reports when no code/file/terminal execution is requested.
- Keep decoding on L0 for questions about this model's identity, official name, parameter scale, base model, LoRA/expert composition, architecture, or post-training foundation, including requests that correct an inaccurate identity claim.
- Keep decoding on L0 for generic checklists, critiques, rubrics, report reviews, concise overviews, benchmark-performance summaries, naming ideas, high-level comparisons, and study-style explanations.
- Keep decoding on L0 when words like concise, report, checklist, performance, guide, guidance, write, or review are used in a general-chat sense and the request does not ask for code, local files, UI artifacts, or terminal execution.
- Route away from L0 only when the request clearly needs one of the specialist task families: L1 personal-agent/customer-service workflows, L2 coding/terminal workflows, or L3 A2UI/UI-generation workflows.
- If multiple specialist routes look equally plausible and no route has a clear task-family signal, choose L0.
